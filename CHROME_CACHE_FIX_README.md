# Исправление проблем с отображением в Chrome

## Проблема
Приложение TenderOptima корректно отображается в Edge и приватном режиме, но показывает проблемы в Chrome из-за агрессивного кэширования.

## Причины
1. **Агрессивное кэширование Chrome** - браузер сохраняет старые версии файлов
2. **Service Workers** - могут быть зарегистрированы устаревшие service workers
3. **Кэш приложения** - Chrome сохраняет старые версии JavaScript/CSS
4. **Куки и локальное хранилище** - могут содержать устаревшие данные

## Решения

### 🚀 Быстрое решение (рекомендуется)
```powershell
# Запустите скрипт для автоматического исправления
.\fix-chrome-display.ps1
```

### 🧹 Полная очистка кэша
```powershell
# Запустите скрипт для полной очистки кэша Chrome
.\clear-chrome-cache.ps1
```

### 🔧 Ручные действия в Chrome

#### 1. Жесткая перезагрузка
- Нажмите `Ctrl + Shift + R`
- Или щелкните правой кнопкой по кнопке обновления и выберите "Очистить кэш и жесткая перезагрузка"

#### 2. Очистка данных сайта
1. Откройте `chrome://settings/content/all`
2. Найдите `localhost:5000`
3. Нажмите на иконку корзины для удаления данных

#### 3. Очистка всех данных браузера
1. Откройте `chrome://settings/clearBrowserData`
2. Выберите "Все время"
3. Отметьте все пункты
4. Нажмите "Очистить данные"

#### 4. Отключение аппаратного ускорения
1. Откройте `chrome://flags/`
2. Найдите "Hardware-accelerated video decode"
3. Установите в "Disabled"
4. Перезапустите Chrome

### 🔍 Диагностика проблем

#### Проверка консоли разработчика
1. Нажмите `F12`
2. Перейдите на вкладку **Console**
3. Ищите ошибки (красный текст)
4. Перейдите на вкладку **Network**
5. Обновите страницу и проверьте статус загрузки файлов

#### Проверка Service Workers
1. Откройте `chrome://serviceworker-internals/`
2. Найдите localhost:5000
3. Нажмите "Unregister" для удаления

### ⚙️ Технические изменения

Внесены следующие изменения для предотвращения проблем с кэшированием:

#### 1. Заголовки сервера
- Добавлены агрессивные заголовки `Cache-Control`
- Настроены `ETag` и `Last-Modified` заголовки
- Добавлены заголовки безопасности

#### 2. Версионирование ресурсов
- Добавлены временные метки к скриптам
- Настроено хэширование файлов в production
- Добавлены мета-теги для предотвращения кэширования

#### 3. Конфигурация Vite
- Настроено хэширование имен файлов
- Добавлены заголовки кэширования для development
- Оптимизирована конфигурация сборки

### 🎯 Альтернативные решения

#### 1. Использование режима инкогнито
- Откройте Chrome в режиме инкогнито (`Ctrl + Shift + N`)
- Перейдите на `http://localhost:5000`

#### 2. Использование другого браузера
- Edge (работает корректно)
- Firefox
- Safari

#### 3. Переустановка Chrome
- Удалите Chrome полностью
- Переустановите последнюю версию

### 📞 Если проблема остается

1. **Проверьте версию Chrome** - обновите до последней версии
2. **Отключите расширения** - запустите Chrome без расширений
3. **Проверьте антивирус** - временно отключите для тестирования
4. **Обратитесь к разработчику** - предоставьте скриншоты консоли разработчика

### 🔄 Профилактика

Для предотвращения подобных проблем в будущем:

1. **Регулярно очищайте кэш** - используйте скрипты автоматизации
2. **Используйте версионирование** - все статические ресурсы должны иметь хэши
3. **Мониторьте консоль** - регулярно проверяйте ошибки в консоли разработчика
4. **Тестируйте в разных браузерах** - убедитесь в совместимости

---

**Примечание**: Данные изменения оптимизированы для режима разработки. В production режиме кэширование настроено более агрессивно для лучшей производительности.
