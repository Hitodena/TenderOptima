# 🚀 План поэтапного внедрения исправлений безопасности

## 📋 Общий план

### Этап 1: Критические исправления (Приоритет: ВЫСОКИЙ)
**Время выполнения: 2-3 часа**
**Риск: НИЗКИЙ** (только исправления безопасности)

#### 1.1 Исправление слабых секретных ключей
- [ ] Создать резервную копию: `.\backup.ps1 v1-secrets`
- [ ] Изменить `server/auth.ts`:
  ```typescript
  // БЫЛО:
  const TOKEN_SECRET = process.env.TOKEN_SECRET || 'supplier-finder-token-secret';
  
  // СТАЛО:
  const TOKEN_SECRET = process.env.TOKEN_SECRET;
  if (!TOKEN_SECRET && process.env.NODE_ENV === 'production') {
    throw new Error('TOKEN_SECRET must be set in production');
  }
  ```
- [ ] Протестировать: `npm run dev`
- [ ] Создать .env.example с примером

#### 1.2 Отключение DEV_MODE в продакшене
- [ ] Изменить логику DEV_MODE:
  ```typescript
  // БЫЛО:
  const DEV_MODE = process.env.DEV_MODE === 'true';
  
  // СТАЛО:
  const DEV_MODE = process.env.NODE_ENV !== 'production' && process.env.DEV_MODE === 'true';
  ```
- [ ] Протестировать в dev и production режимах

#### 1.3 Добавление rate limiting
- [ ] Установить пакет: `npm install express-rate-limit`
- [ ] Добавить в `server/index.ts`:
  ```typescript
  import rateLimit from 'express-rate-limit';
  
  const authLimiter = rateLimit({
    windowMs: 15 * 60 * 1000, // 15 минут
    max: 5, // максимум 5 попыток
    message: 'Слишком много попыток входа'
  });
  
  app.use('/api/auth/login', authLimiter);
  ```

#### 1.4 Исправление CORS настроек
- [ ] Ограничить CORS:
  ```typescript
  app.use(cors({
    origin: process.env.ALLOWED_ORIGINS?.split(',') || ['http://localhost:3000'],
    credentials: true
  }));
  ```
