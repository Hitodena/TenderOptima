# 🔄 План управления версиями и откатов

## 📁 Структура резервных копий

```
backup/
├── auth-backup/
│   ├── auth-original.ts          # Оригинальная версия auth.ts
│   ├── auth-v1-secure.ts         # Версия с исправлениями безопасности
│   └── auth-v2-enhanced.ts       # Версия с дополнительными улучшениями
├── middleware-backup/
│   ├── requireAuth-original.ts   # Оригинальная версия middleware
│   └── requireAuth-secure.ts     # Безопасная версия
├── schema-backup/
│   ├── schema-original.ts        # Оригинальная схема БД
│   └── schema-migrations/        # Миграции схемы
├── config-backup/
│   ├── package-original.json     # Оригинальный package.json
│   └── env-templates/            # Шаблоны .env файлов
└── README.md                     # Этот файл
```

## 🔄 Процедура отката

### Быстрый откат (критическая ситуация)
```bash
# Откат auth.ts
copy backup\auth-backup\auth-original.ts server\auth.ts

# Откат middleware
copy backup\middleware-backup\requireAuth-original.ts server\middleware\requireAuth.ts

# Перезапуск сервера
npm run dev
```

### Полный откат с миграцией БД
```bash
# 1. Откат кода
copy backup\auth-backup\auth-original.ts server\auth.ts

# 2. Откат схемы БД (если нужно)
copy backup\schema-backup\schema-original.ts shared\schema.ts

# 3. Пересоздание миграций
npm run db:generate

# 4. Применение миграций
npm run db:push

# 5. Перезапуск
npm run dev
```

## 🚨 Экстренные процедуры

### Если сервер не запускается
1. Проверить логи: `npm run dev`
2. Откатить к предыдущей версии
3. Проверить зависимости: `npm install`
4. Очистить кэш: `npm run build`

### Если БД повреждена
1. Остановить сервер
2. Откатить схему: `copy backup\schema-backup\schema-original.ts shared\schema.ts`
3. Пересоздать миграции: `npm run db:generate`
4. Применить миграции: `npm run db:push`

## 📝 Логирование изменений

Каждое изменение должно быть задокументировано:
- Дата и время
- Описание изменения
- Файлы, которые были изменены
- Тестирование
- Результат

## 🔐 Безопасность резервных копий

- Резервные копии содержат чувствительные данные
- Не коммитить в Git
- Хранить в безопасном месте
- Регулярно обновлять
