# 🔄 Логика извлечения параметров из email

## 📋 Правила приоритизации данных

### 1️⃣ **ПРИОРИТЕТ ВЛОЖЕНИЙ** 
```
Если данные найдены ВО ВЛОЖЕНИИ:
├── ✅ Используем данные из вложения
└── ❌ Игнорируем данные из тела письма (даже если они есть)
```

### 2️⃣ **ДОПОЛНЕНИЕ ИЗ ТЕЛА ПИСЬМА**
```
Если данные НЕ найдены во вложении:
├── ✅ Используем данные из тела письма
└── ❌ Если и в теле письма нет - ставим "-"
```

### 3️⃣ **ЧАСТИЧНОЕ ДОПОЛНЕНИЕ**
```
Если во вложении найдены НЕ ВСЕ параметры:
├── ✅ Найденные во вложении - используем из вложения
├── ✅ НЕ найденные во вложении - берем из тела письма
└── ❌ Если нигде не найдено - ставим "-"
```

## 🎯 Примеры работы логики

### Пример 1: Полные данные во вложении
```
Вложение: "Цена: 100500 руб, Количество: 1000 шт"
Тело письма: "Цена: 95000 руб, Количество: 800 шт"

Результат:
├── Цена: 100500 руб (из вложения) ✅
└── Количество: 1000 шт (из вложения) ✅
```

### Пример 2: Данные только в теле письма
```
Вложение: "См. коммерческое предложение в приложении"
Тело письма: "Цена: 100500 руб, Количество: 1000 шт"

Результат:
├── Цена: 100500 руб (из тела письма) ✅
└── Количество: 1000 шт (из тела письма) ✅
```

### Пример 3: Частичные данные во вложении
```
Вложение: "Цена: 100500 руб"
Тело письма: "Количество: 1000 шт, Срок поставки: 2 недели"

Результат:
├── Цена: 100500 руб (из вложения) ✅
├── Количество: 1000 шт (из тела письма) ✅
└── Срок поставки: 2 недели (из тела письма) ✅
```

### Пример 4: Нет данных нигде
```
Вложение: "См. коммерческое предложение в приложении"
Тело письма: "Спасибо за запрос, ответим в ближайшее время"

Результат:
├── Цена: - (не найдено) ❌
├── Количество: - (не найдено) ❌
└── Срок поставки: - (не найдено) ❌
```

## 🔧 Техническая реализация

### Код логики:
```typescript
parameters.forEach(paramName => {
  // 1. ПРИОРИТЕТ: Если данные есть во вложении - используем их
  if (hasAttachmentData && attachmentParameters[paramName] && 
      attachmentParameters[paramName] !== '-' && 
      attachmentParameters[paramName] !== '' && 
      attachmentParameters[paramName] !== null) {
    finalParameters[paramName] = String(attachmentParameters[paramName]);
    console.log(`Priority rule: Using attachment value for ${paramName}: ${attachmentParameters[paramName]}`);
  } 
  // 2. ДОПОЛНЕНИЕ: Если во вложении данных нет, но есть в теле письма - используем из тела
  else if (bodyParameters[paramName] && 
           bodyParameters[paramName] !== '-' && 
           bodyParameters[paramName] !== '' && 
           bodyParameters[paramName] !== null) {
    finalParameters[paramName] = String(bodyParameters[paramName]);
    console.log(`Priority rule: Using body value for ${paramName}: ${bodyParameters[paramName]} (attachment data not found)`);
  } 
  // 3. НЕТ ДАННЫХ: Если нигде не найдено
  else {
    finalParameters[paramName] = '-';
    console.log(`Priority rule: No value found for ${paramName}, using default '-'`);
  }
});
```

## ✅ Проверка корректности

**Да, логика работает именно так, как вы описали:**

1. ✅ **Приоритет вложений** - если данные найдены во вложении, они имеют приоритет
2. ✅ **Дополнение из тела** - если во вложении данных нет, берем из тела письма  
3. ✅ **Частичное дополнение** - если во вложении найдены не все параметры, недостающие берем из тела письма
4. ✅ **Fallback** - если нигде не найдено, ставим "-"

Эта логика реализована в обоих IMAP сервисах:
- `server/imap-service.ts` (основной)
- `server/imap-service-personal.ts` (персональный)
