## Оценка бизнес-приоритетов (ОБЯЗАТЕЛЬНО)

### 1. Какие задачи бизнеса вы хотите решить с помощью цифровых услуг компании А1? Новый проект или оптимизация работы существующего?

**Новый проект** — запуск и промышленная эксплуатация SaaS-платформы TenderOptima для автоматизации закупочной деятельности B2B-клиентов.

**Решаемые задачи:**
- автоматический поиск поставщиков по запросу и региону;
- массовая рассылка запросов коммерческих предложений (КП);
- ведение переписки с поставщиками в едином интерфейсе (SMTP/IMAP);
- AI-извлечение требований из технических заданий (ТЗ) и сравнение КП с ТЗ;
- экспорт результатов (XLSX/DOCX);
- приём заявок с сайта (лидогенерация, мини-CRM);
- биллинг и подписки для корпоративных клиентов.

Продукт находится в стадии вывода в промышленную эксплуатацию. Цель обращения в А1 — размещение информационной системы в **аттестованном сегменте ЦОД** для законной обработки персональных и конфиденциальных данных.

### 2. Какие сервисы планируется запускать или переносить в облако?

| Сервис | Технология | Назначение |
|--------|------------|------------|
| Веб-приложение (сайт + личный кабинет) | Nuxt 4 (SSR), порт 443/3000 | Интерфейс пользователя |
| REST API | FastAPI (Python 3.12), порт 8000 | Бизнес-логика |
| Сервис парсинга | FastAPI, порт 8010 | Поиск поставщиков (Yandex Search API) |
| Фоновые задачи | Celery Worker + Celery Beat | Рассылка, IMAP-опрос, AI-анализ, уведомления |
| СУБД | PostgreSQL 18 | Хранение данных |
| Кэш и очереди | Redis 8 | Очереди Celery, кэш |
| Файловое хранилище | Локальный том (Docker volume) | ТЗ, КП, вложения из почты |

**Развёртывание:** Docker Compose на Linux (7 постоянных контейнеров: db, redis, backend, parser, worker, beat, frontend).

**Не планируется на первом этапе:** 1С, ERP, Kubernetes.

**Внешние интеграции (исходящий HTTPS):** LLM API (OpenAI/DeepSeek), Yandex Search API, SMTP/IMAP почтовых серверов клиентов.

### 6. Как происходит резервное копирование? (BaaS)

**Сейчас:** резервное копирование в продакшене не автоматизировано.

**Требуется от А1 (BaaS):**
- ежедневное резервное копирование PostgreSQL (полное + непрерывная защита/WAL при наличии);
- ежедневное резервное копирование файлового хранилища (том `uploads`);
- хранение копий: оперативные — **30 дней**, полные еженедельные — **90 дней**;
- хранение резервных копий в отдельном от основного ЦОД контуре;
- **RPO:** 24 часа;
- **RTO:** 4 часа;
- тестовое восстановление — **1 раз в квартал**.

## Технические детали и требования

### 7. Какие типы виртуальных машин, объёмы хранения и сетевые ресурсы вам нужны?

**Горизонт планирования:** 12 месяцев, сценарий роста.
**Окружения:** одна prod-ВМ в ЦОД; staging в облаке **не требуется** (тестирование локально у разработчика).

#### Целевая нагрузка (12-й месяц)

| Показатель | Значение |
|------------|----------|
| B2B-компании / пользователи | 300 / 300 |
| Одновременно в пике | 30 |
| Закупок в месяц | 300 |
| Поставщиков на закупку (ср. / макс.) | 10 / 100 |
| AI-анализ ТЗ в месяц | 300 |
| Сравнение КП с ТЗ в месяц | 1 500 |
| Парсер: поисков в день (пик) | 200 (пик 30/час) |
| Исходящие письма | 450 000 / мес (~20 500 / раб. день) |
| Входящие письма (только цепочки с поставщиками по `[TID-…]`) | ~675 000 / мес (~30 700 / раб. день) |
| Хранение данных после ухода клиента | 1 год, затем удаление |

#### Почтовая интеграция

Точечная: система **хранит только переписку с поставщиками** по закупкам, идентифицируемую меткой `[TID-…]` в теме письма. Полная синхронизация корпоративного почтового ящика **не выполняется**.

#### Конфигурация prod-ВМ (поэтапно)

| Период | vCPU | RAM | SSD |
|--------|------|-----|-----|
| Старт (1–3 мес) | 4 | 16 ГБ | 200 ГБ |
| Рост (4–6 мес) | 8 | 32 ГБ | 500 ГБ |
| Целевая (12 мес) | **16** | **64 ГБ** | **1,8 ТБ NVMe** |

**На одной prod-ВМ (Docker Compose):** PostgreSQL, Redis, Nuxt frontend, FastAPI backend, Celery worker/beat, parser.

**Распределение RAM (целевая конфигурация):**
- PostgreSQL — 16–24 ГБ
- Celery workers (почта + AI) — 24–32 ГБ
- Backend, parser, frontend — 8 ГБ
- Redis — 4–8 ГБ
- ОС и буфер — 8 ГБ

**Celery:** 16 worker-процессов (10 — почта, 4 — AI, 2 — parser).

#### Объёмы хранения (12-й месяц)

| Компонент | Объём |
|-----------|-------|
| PostgreSQL (письма, закупки, пользователи) | ~250–300 ГБ |
| Файлы (ТЗ, КП, вложения) | ~600 ГБ – 1,2 ТБ |
| ОС, Docker, логи, запас | ~200 ГБ |
| **Итого SSD** | **~1,8 ТБ** |

#### Сетевые требования

- публичный IP, **HTTPS (443)** для `tenderoptima.online`;
- исходящий: SMTP (25/465/587), IMAP (993), HTTPS к LLM API и Yandex API;
- ориентировочный исходящий трафик: **200–400 ГБ/мес**;
- администрирование: SSH (доступ по ключу, ограничение по IP — желательно).

### 9. Какая у вас внутренняя инфраструктура и как планируется интеграция с облаком?

**Внутренняя инфраструктура:**
- разработка — локальные машины + GitHub;
- продакшен — одна ВМ в ЦОД А1;
- офисный доступ администраторов — через SSH (VPN — по согласованию с А1).

- LLM и Yandex Search — исходящие HTTPS-запросы из worker/parser.

**Каналы связи:** стандартный интернет; для администрирования — SSH с ограничением по IP.
