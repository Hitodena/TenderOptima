План кардинального улучшения TenderOptima
Сайт: tenderoptima.online
Эталон: trafory.com
Стек: Nuxt 4 / Vue 3 / TypeScript / NuxtUI v4 / Tailwind v4 (фронт) · FastAPI / Python 3.12 / Celery / Redis / PostgreSQL 18 / SQLAlchemy 2 / LLM (OpenAI, DeepSeek) (бэк)
Главная проблема сегодня. Сайт «рассказывает» про продукт текстом, но не «показывает» его. Центрированный hero, монотонно-синяя палитра, абстрактные блобы вместо интерфейса, единственная CTA — «Регистрация», нет ни одного доверительного блока, контактов, кейсов или тарифов. У Trafory наоборот: реальный скриншот продукта, trust-бейджи, многократные CTA, понятная структура и формы заявок по всей странице.
Ключевая установка. Главный экран продаёт заявку, а не регистрацию. Кнопку «Регистрация» убираем из навигации и hero полностью — вход в систему происходит только после того, как менеджер обработал лид и подключил клиента. Всё работает через заявки → мини-CRM.

1. ВИЗУАЛЬНО
1.1. Hero — асимметричный layout + продуктовый proof
Вместо центрированного стека: слева текстовый блок (заголовок-обещание + подзаголовок + 2 CTA + trust-строка), справа — реальный скриншот/макет интерфейса TenderOptima (три колонки: список поставщиков → переписка → таблица соответствия ТЗ/КП).
Заголовок — не «Поиск поставщиков», а обещание результата: «Сократите ручной анализ КП и ТЗ в 10 раз». Подзаголовок проясняет: поиск, рассылка, ответы, AI-сравнение — в одном браузерном интерфейсе, без ручной сверки таблиц.
CTA: «Оставить заявку» (primary) + «Получить демо» (secondary). Никакой «Регистрации».
Снизу под hero — тонкая строка доверия: «Работает в браузере · Без установки · ×10 к скорости · до 95% точность · -80% рисков».
1.2. Палитра и глубина
Уйти от монотонного синего. Белый фон + один фирменный акцент (тёмно-синий/инди как база) + второй акцент-крюк (тёплый оранжевый или бирюза) для CTA и статусов — как контраст, ведущий взгляд.
Убрать «плавающие блобы». Заменить на: продуктовый скриншот, плавающие «облака»-плашки с цифрами/бейджами (как у Trafory), микро-анимации появления секций (Vue Transition + Intersection Observer).
Единая система отступов и сетки (Tailwind v4 design tokens), типографическая шкала с чёткой иерархией (display / h1 / h2 / body / caption).
1.3. Trust-блоки (критично для B2B)
Trafory сильна доверием. Внедрить по аналогии, адаптировав к закупкам:
Безопасность данных — где хранятся ТЗ и КП (РФ/ЕАЭС контур), шифрование, роли доступа, NDA.
Как подключаемся к почте — SMTP/IMAP безопасно, права, что мы не храним пароли в открытом виде.
Закрытый контур / on-premise — возможность развернуть у клиента (у вас уже Docker Compose — это плюс для Enterprise).
Бейджи: резидентство, аттестация ПО (если есть/планируется), 16+ лет экспертизы — поставьте реальные цифры; если их нет — пока не пишите.
Логотипы клиентов / отрасли («работаем с производством, ритейлом, IT, энергетикой…»).
1.4. Мультиплексные CTA
Повторяющиеся призывы по всей странице (как у Trafory): «Получить консультацию», «Запросить демонстрацию», «Обратный звонок». Каждая ведёт в единую форму заявки (см. раздел 3.1).
1.5. Качество мелочей
Реальные скриншоты интерфейса вместо абстрактной графики на всех секциях «Как это работает».
Иконки единым стилем (NuxtUI icon set), без смеси наборов.
Анимации статичные-при-скролле, без перегрузки.
Адаптив и mobile-first: у вас NuxtUI/Tailwind — проверить все брейкпоинты, особенно асимметричный hero.

3. МОДУЛИ
3.1. Модуль заявок (лидогенерация) — ядро конверсии
Это не просто форма, а мини-CRM на бэке (FastAPI + PostgreSQL, у вас всё для этого есть):
Форма заявки — поля: имя, компания, email, телефон, роль/ICP, что интересует, комментарий. + согласие на обработку ПДн (обязательно для ЕАЭС/РФ).
Захват UTM — источник, кампания, utm_source/medium/content/term, страница, referrer, первый/последний клик.
Лиды в БД — таблица leads со статусами (новый → в работе → демо → квалифицирован → клиент → отказ).
Админка лидов — список, фильтры, поиск, смена статуса, комментарии менеджера, история касаний, экспорт в XLSX.
Уведомления — на email менеджера и в Telegram-бота при новом лиде (у вас уже есть SMTP/IMAP и Telegram-стек).
Авто-ответ заявителю — подтверждение приёма заявки на email.
SLA реакции — индикатор «свяжемся в течение N часов».
Фронт: переиспользуемый компонент формы (Nuxt), встраивается в hero-CTA, секции, финальную зону, страницу «Контакты» — все пишут в один эндпоинт.
