# Отчет об исправлении извлечения цен

## Проблема
AI промпт неправильно заполнял данные с ценами, извлекая ИНН (7713471291) как цены вместо правильных значений из документа:
- **Неправильно**: 7,713,471,291 руб. (ИНН)
- **Правильно**: 128,000 руб. (общая стоимость), 0,16 руб/1эт (цена за единицу)

## Найденные проблемы

### 1. AI промпт не различал цены и номера
- ИНН (7713471291) извлекался как цена
- Отсутствовала валидация числовых значений
- Промпт не содержал четких инструкций по разграничению

### 2. Отсутствие валидации в коде
- Функция `extractParameterFromText` не проверяла валидность цен
- Принимались любые числовые значения как цены

## Внесенные исправления

### 1. Улучшен AI промпт в `server/routes/extract-parameters.ts`
```typescript
КРИТИЧЕСКИ ВАЖНО - РАЗЛИЧЕНИЕ ЦЕН И НОМЕРОВ:
- НЕ ИСПОЛЬЗУЙ ИНН, ОГРН, КПП, номера телефонов как цены!
- ИНН обычно 10-12 цифр подряд (например: 7713471291)
- ОГРН обычно 13-15 цифр подряд
- КПП обычно 9 цифр подряд
- Цены обычно меньше и содержат запятые или точки (например: 128000, 0,16, 320)
- Цены часто сопровождаются словами "руб", "рублей", "₽"
```

### 2. Улучшен AI промпт в `server/services/deepseek-api.ts`
Добавлены аналогичные инструкции по разграничению цен и номеров.

### 3. Добавлена валидация цен в `extractParameterFromText`
```typescript
// Function to validate if a number looks like a price (not INN, OGRN, etc.)
const isValidPrice = (value: string): boolean => {
  const cleanValue = value.replace(/[^\d.,]/g, '');
  const numValue = parseFloat(cleanValue.replace(',', '.'));
  
  // Reject very large numbers that look like INN/OGRN (10+ digits)
  if (cleanValue.length >= 10) {
    return false;
  }
  
  // Reject numbers that are too small to be meaningful prices (less than 0.01)
  if (numValue < 0.01) {
    return false;
  }
  
  // Accept reasonable price ranges
  return numValue >= 0.01 && numValue <= 10000000; // Up to 10 million
};
```

### 4. Обновлена логика извлечения цен
- Добавлена проверка `isValidPrice()` для всех найденных числовых значений
- Отклоняются значения, похожие на ИНН/ОГРН (10+ цифр)
- Принимаются только разумные диапазоны цен (0.01 - 10,000,000)

## Результат
Теперь система будет:
- ✅ Правильно различать цены и номера документов
- ✅ Извлекать корректные цены из документов
- ✅ Отклонять ИНН/ОГРН как цены
- ✅ Валидировать числовые значения перед извлечением
