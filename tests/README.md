# 🧪 Тестирование SupplierFinder

## 📁 Структура тестирования

```
tests/
├── setup/                 # Настройка тестового окружения
│   ├── test-environment.ts
│   ├── test-database.ts
│   └── test-config.ts
├── unit/                  # Модульные тесты
│   ├── api/              # API endpoints
│   ├── components/       # React компоненты
│   ├── services/         # Бизнес-логика
│   └── utils/            # Утилиты
├── integration/          # Интеграционные тесты
│   ├── database/         # База данных
│   ├── email/           # Email сервисы
│   ├── ai/              # AI интеграции
│   └── external/        # Внешние API
├── e2e/                 # End-to-End тесты
│   ├── user-journey/    # Пользовательские сценарии
│   ├── critical-paths/  # Критические пути
│   └── ui/              # UI тестирование
├── performance/         # Тесты производительности
│   ├── load/            # Нагрузочное тестирование
│   ├── stress/          # Стресс-тестирование
│   └── scalability/    # Масштабируемость
├── security/            # Тесты безопасности
│   ├── auth/            # Аутентификация
│   ├── authorization/   # Авторизация
│   ├── csrf/           # CSRF защита
│   └── injection/      # SQL injection, XSS
└── utils/               # Утилиты для тестирования
    ├── helpers/         # Вспомогательные функции
    ├── mocks/          # Моки и стабы
    └── fixtures/       # Тестовые данные
```

## 🚀 Запуск тестов

```bash
# Все тесты
npm run test

# Модульные тесты
npm run test:unit

# Интеграционные тесты
npm run test:integration

# E2E тесты
npm run test:e2e

# Тесты производительности
npm run test:performance

# Тесты безопасности
npm run test:security

# Покрытие кода
npm run test:coverage
```
