# Улучшение алгоритма авто проверки email

## Проблема
Ранее алгоритм авто проверки входящих писем (когда `ENABLE_EMAIL_CHECKING=true`) проверял email аккаунты всех пользователей, у которых было заполнено поле `emailAccount`, но не проверял наличие `emailPassword`. Это могло приводить к ошибкам при попытке подключения к IMAP серверам без паролей.

## Решение
Модифицирована функция `getUsersWithEmailConfig()` в файле `server/storage.ts` для проверки **ОБОИХ** полей:
- `emailAccount` - должен быть заполнен и не пустой
- `emailPassword` - должен быть заполнен и не пустой

## Изменения

### 1. server/storage.ts
```typescript
// БЫЛО:
.where(and(
  eq(users.emailConfigured, true),
  isNotNull(users.emailAccount)
));

// СТАЛО:
.where(and(
  eq(users.emailConfigured, true),
  isNotNull(users.emailAccount),
  isNotNull(users.emailPassword)  // ← ДОБАВЛЕНО
));
```

### 2. server/index.ts
Улучшено логирование для отладки:
- Показывает количество пользователей с полной конфигурацией
- Выводит информацию о пользователях с частичной конфигурацией
- Помогает понять, почему некоторые пользователи не включены в проверку

## Результат
Теперь авто проверка email будет выполняться **ТОЛЬКО** для пользователей, у которых заполнены **ОБА** поля:
- ✅ `emailAccount` - адрес электронной почты
- ✅ `emailPassword` - пароль для доступа к почтовому ящику

## Логирование
При запуске авто проверки в логах будет видно:
```
📧 Found X users with BOTH emailAccount AND emailPassword configured
📧 Note: Y users have emailAccount but missing emailPassword: [список пользователей]
```

## Тестирование
Для проверки работы алгоритма можно:
1. Установить `ENABLE_EMAIL_CHECKING=true` в `.env`
2. Запустить сервер
3. Проверить логи - должны показывать только пользователей с полной конфигурацией

## Совместимость
Изменения полностью обратно совместимы и не влияют на существующую функциональность.
