# Исправления модального окна "Запрос на улучшение условий"

## Исправленные проблемы

### 1. ✅ Исправлена отправка письма

**Проблема**: API запрос отправлял неправильные параметры, что приводило к ошибке "Не удалось отправить запрос"

**Решение**: 
- Изменил параметры API запроса с `supplierId` на правильные `supplierEmail` и `supplierName`
- Теперь запрос соответствует ожиданиям серверного API `/api/improvement-request`

**Было**:
```javascript
{
  supplierId: supplier.email,
  requestId,
  subject: subject.trim(),
  message: message.trim()
}
```

**Стало**:
```javascript
{
  requestId,
  supplierEmail: supplier.email,
  supplierName: supplier.name,
  subject: subject.trim(),
  message: message.trim()
}
```

### 2. ✅ Исправлена высота секции шаблонов

**Проблема**: При создании большого количества шаблонов высота модального окна увеличивалась, что было неудобно

**Решение**:
- Убрал фиксированную минимальную высоту `min-h-[400px]` из правой колонки
- Добавил `max-h-[400px]` для списка шаблонов с автоматическим скроллом
- Сделал заголовок секции шаблонов `flex-shrink-0` чтобы он не сжимался

**Изменения в компонентах**:

#### `improved-improvement-request-modal.tsx`:
- Убрал `min-h-[400px]` из правой колонки
- Добавил `flex flex-col` для правильного layout

#### `email-template-manager.tsx`:
- Убрал `min-h-[350px]` из основного контейнера
- Добавил `max-h-[400px]` для списка шаблонов
- Сделал заголовок `flex-shrink-0`

## Результат

1. **Отправка писем работает корректно** - API получает правильные параметры и письма отправляются без ошибок
2. **Высота модального окна стабильна** - независимо от количества шаблонов, окно сохраняет фиксированную высоту
3. **Удобный скролл** - при большом количестве шаблонов появляется скролл только в секции шаблонов, не влияя на общую высоту окна

## Технические детали

- API endpoint: `/api/improvement-request`
- Максимальная высота списка шаблонов: 400px
- Скролл появляется автоматически при превышении лимита
- Заголовок секции шаблонов остается видимым при скролле
