# Отчет об исправлении автоматического извлечения параметров

## Проблема
При открытии нового письма в разделе "Извлеченные параметры" отображались только данные из тела письма, но не из вложений (например, PDF). При ручном нажатии кнопки "Обновить изв. параметры" все работало корректно.

## Причина
Автоматическое извлечение параметров на бэкенде происходило до полной обработки вложений. Между анализом вложений и извлечением параметров была фиксированная задержка в 3 секунды, что могло быть недостаточно для обработки больших PDF файлов.

## Исправление
Добавлена логика ожидания полной обработки вложений перед извлечением параметров в файлах:
- `server/imap-service.ts` (строки 1102-1126)
- `server/imap-service-personal.ts` (строки 789-813)

### Изменения:
1. **Замена фиксированной задержки на активное ожидание:**
   ```typescript
   // Было:
   await new Promise(resolve => setTimeout(resolve, 3000));
   
   // Стало:
   if (attachments && attachments.length > 0) {
     let attempts = 0;
     const maxAttempts = 10; // Максимум 30 секунд ожидания
     
     while (attempts < maxAttempts) {
       const updatedResponse = await storage.getSupplierResponseById(response.id);
       if (updatedResponse && updatedResponse.attachments && Array.isArray(updatedResponse.attachments)) {
         const hasProcessedAttachments = updatedResponse.attachments.some((att: any) => att.extractedText);
         if (hasProcessedAttachments) {
           console.log(`✅ Attachments processed for response ${response.id}`);
           break;
         }
       }
       
       await new Promise(resolve => setTimeout(resolve, 3000));
       attempts++;
     }
   }
   ```

2. **Проверка типизации для безопасности:**
   - Добавлена проверка `Array.isArray(updatedResponse.attachments)` для предотвращения ошибок типизации

## Как это работает
1. При получении письма запускается автоматическое извлечение параметров
2. Сначала анализируются вложения (если есть)
3. **НОВОЕ:** Система ждет, пока вложения полностью обработаются и сохранятся в базу данных
4. Только после этого извлекаются параметры из тела письма и вложений
5. Применяются правила приоритета: вложения > тело письма

## Результат
Теперь при открытии письма пользователь сразу видит полные извлеченные параметры, включая данные из вложений, без необходимости ручного обновления.
