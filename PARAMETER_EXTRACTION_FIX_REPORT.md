# Отчет об исправлении логики извлечения параметров

## Проблема

Система не извлекала параметры из тела письма, когда вложения присутствовали, но не содержали полезной информации. В интерфейсе отображалось "В данном емайле нет параметров для извлечения", хотя в теле письма была важная информация типа "Общая стоимость без НДС: 50001 BYN".

## Анализ проблемы

### Корневая причина
1. **Приоритизация вложений**: Система сначала проверяла вложения, и только при полном отсутствии результата переходила к тексту письма
2. **Недостаточная проверка качества вложений**: Система не различала случаи, когда вложение есть, но не содержит полезного текста
3. **Слабая fallback логика**: Отсутствовала надежная логика возврата к анализу тела письма

### Технические детали
- Вложения с изображениями могли содержать пустой или ошибочный `extractedText`
- Система считала наличие вложения достаточным основанием для игнорирования тела письма
- Отсутствовала проверка качества извлеченного из вложений текста

## Внесенные исправления

### 1. Улучшенная логика проверки вложений

**Файл:** `server/routes/extract-parameters.ts`

**Изменения:**
```typescript
// Добавлена проверка качества содержимого вложений
const extractedText = attachment.extractedText.trim();
if (extractedText.length < 10 || 
    extractedText.includes('Error extracting') || 
    extractedText.includes('Ошибка') ||
    extractedText.includes('No text found')) {
  console.log(`Attachment ${attachment.filename} has no meaningful content, skipping`);
  continue;
}
```

**Результат:** Система теперь пропускает вложения с некачественным содержимым.

### 2. Улучшенная fallback логика

**Изменения:**
```typescript
// Use email content if:
// 1. No attachment result was found (bestResult.value === "-")
// 2. OR attachment result has very low confidence (< 0.3)
// 3. OR attachment result is empty/meaningless
if ((bestResult.value === "-" || bestResult.confidence < 0.3 || bestResult.value.trim() === "") && contentResult.value !== "-") {
  bestResult = contentResult;
  console.log(`Using email content for parameter ${parameter}: ${bestResult.value}`);
}
```

**Результат:** Система теперь использует текст письма в более широком диапазоне случаев.

### 3. Альтернативная функция извлечения стоимости

- Автоматическое определение валюты
- Fallback для случаев, когда основная функция не сработала


## Логика работы после исправлений

### Алгоритм извлечения параметров:

1. **Проверка вложений:**
   - Проверяет наличие вложений
   - Анализирует качество извлеченного текста
   - Пропускает вложения с некачественным содержимым
   - Отмечает, если ни одно вложение не содержит полезной информации

2. **Fallback к тексту письма:**
   - Используется, если вложения не дали результата
   - Используется, если результат из вложений имеет низкую уверенность (< 0.3)
   - Используется, если результат из вложений пустой

3. **Альтернативное извлечение:**
   - Применяется для критически важных параметров (стоимость)
   - Использует более агрессивные паттерны
   - Работает как дополнительный fallback
