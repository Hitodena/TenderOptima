# Обновление полей в админ панели подписок

## Задача
Обновить форму редактирования подписки в админ панели:
- ✅ Добавить новые поля для системного логина
- ✅ Переименовать существующие поля для email
- ✅ Обновить загрузку данных при редактировании

## Выполненные изменения

### ✅ **1. Обновлен интерфейс SubscriptionFormData**
**Файл**: `client/src/pages/admin/subscriptions-page.tsx`

```typescript
interface SubscriptionFormData {
  userId: number;
  userEmail?: string; // For creating new users
  systemLogin?: string; // System login for user
  systemPassword?: string; // System password for user
  emailAccount?: string; // Email account for user
  emailPassword?: string; // Email password for user
  plan: string;
  status: string;
  maxRequestsPerMonth: number;
  startDate: string;
  endDate: string | null;
  notes: string | null;
}
```

### ✅ **2. Обновлена форма создания подписки**
**Файл**: `client/src/pages/admin/subscriptions-page.tsx`

**Новые поля:**
- **Логин для системы:** - поле для системного логина пользователя
- **Пароль для системы:** - поле для системного пароля пользователя

**Переименованные поля:**
- **"Email аккаунт"** → **"Email аккаунт для емайла:"**
- **"Пароль email"** → **"Пароль для Email аккаунта:"**

### ✅ **3. Обновлена форма редактирования подписки**
**Файл**: `client/src/pages/admin/subscriptions-page.tsx`

Аналогичные изменения применены к форме редактирования с префиксом `edit-` для ID полей.

### ✅ **4. Добавлен серверный endpoint для системных данных**
**Файл**: `server/routes/admin.ts`

```typescript

router.get('/users/:id/system-data', requireAuth, requireAdmin, async (req, res) => {
```

### ✅ **5. Обновлена загрузка данных при редактировании**
**Файл**: `client/src/pages/admin/subscriptions-page.tsx`

Функция `openEditDialog` теперь:

- Загружает email конфигурацию через `/api/admin/users/:id/email-config`
- Заполняет все поля формы актуальными данными

## Результат

### ✅ **Новая структура полей в админ панели:**

#### **Системные данные:**
- **Логин для системы:** - отображает системный логин пользователя (например: `test122@test.com`)
- **Пароль для системы:** - скрыт по соображениям безопасности

#### **Email данные:**
- **Email аккаунт для емайла:** - отображает email для работы с почтой (например: `gmz@tenderoptima.by`)
- **Пароль для Email аккаунта:** - отображает пароль от email аккаунта

### ✅ **Как это работает:**

1. **При создании подписки** - все поля пустые, админ может заполнить их
2. **При редактировании подписки** - поля автоматически заполняются:
   - Системные данные загружаются из таблицы `users`
   - Email данные загружаются из конфигурации пользователя
3. **Безопасность** - системный пароль не отображается, только логин

### ✅ **Пример отображения:**
- **Логин для системы:** `test122@test.com`
- **Пароль для системы:** `[HIDDEN]`
- **Email аккаунт для емайла:** `gmz@tenderoptima.by`
- **Пароль для Email аккаунта:** `actual_email_password`

## Тестирование

1. Откройте админ панель: `http://localhost:5000/admpanel`
2. Перейдите в "Управление подписками"
3. Нажмите "Создать подписку" - проверьте новые поля
4. Нажмите "Редактировать" у существующей подписки - проверьте загрузку данных
